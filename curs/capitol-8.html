Objectes

Els arrays guarden valors ordenats per posició. Els objectes guarden valors per nom: en lloc d'accedir-hi per índex (0, 1, 2…), hi accedeixes amb una etiqueta (nom, edat…). Són ideals per representar coses del món real amb diverses característiques.

Recórrer un objecte

Un objecte no té .length com els arrays. Si vols saber quantes propietats té o recórrer-les, fas servir Object.keys(), Object.values() i Object.entries():

Per recórrer totes les propietats amb un bucle, la manera més habitual és for...in, que itera per les claus:

Llegeix-ho: «per a cada clau dins de persona, imprimeix la clau i el seu valor». Dins del bucle, persona[clau] dona el valor associat — aquí cal usar claudàtors perquè clau és una variable, no una clau literal.

Atenció: for...in (per a objectes) és diferent de for...of (per a arrays). Confondre'ls és un error comú.

Exercici

Tens un array d'objectes que representa alumnes amb el seu nom i la seva nota. Escriu una funció millorAlumne(alumnes) que retorni l'objecte de l'alumne amb la nota més alta.

L'array tindrà sempre almenys un alumne. Per exemple:

[{nom: "Anna", nota: 7}, {nom: "Bernat", nota: 9}, {nom: "Carla", nota: 8}]

Hauria de retornar l'objecte {nom: "Bernat", nota: 9}.

Pista: recorda el patró del màxim del capítol 6. Has de mantenir una variable que apunti al millor alumne trobat fins ara, i actualitzar-la quan en trobis un de millor.

Resum

  • Un objecte agrupa valors amb noms: const p = { nom: "Ada", edat: 16 };.
  • Accedeixes a propietats amb punt (p.nom) o claudàtors (p["nom"]). Els claudàtors són necessaris si la clau és dinàmica.
  • Per afegir, modificar o eliminar propietats: p.x = y, delete p.x.
  • Una propietat inexistent retorna undefined (no dona error).
  • Un objecte no té .length: per saber quantes propietats té, fas Object.keys(obj).length.
  • Recorres un objecte amb for (const clau in obj); un array, amb for (const x of arr).
  • Pots niar objectes dins d'objectes i tenir arrays d'objectes per modelar estructures complexes.
